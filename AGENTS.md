# Theme Workflow

## Add a new theme

1. Check existing theme names first to avoid duplicates or near-duplicates.
   Files:
   - `shared/render-core/enums.ts`
   - `shared/render-core/theme.ts`

2. Add the new enum in `PresetTheme`.
   Rules:
   - Use `snake_case`.
   - Keep the value and enum name aligned, for example `DESERT_MIRAGE = 'desert_mirage'`.

3. Add the theme definition in `shared/render-core/theme.ts`.
   Use one of these helpers:
   - `createTheme(dark, light)` for simple palettes.
   - `createDetailedTheme(dark, light, darkPalette, lightPalette)` for dual-mode themes with custom background/text/grid/stroke metadata.
   - `mirrorTheme(...)` only when dark and light should intentionally be identical.

4. Prefer real dual-mode themes.
   Rules:
   - Provide both `dark` and `light` palettes.
   - Keep palette length consistent between dark and light.
   - Order colors from empty/lowest level to strongest level.
   - Avoid adding a theme that is visually too close to an existing one.

5. Do not manually wire the theme into the config panel.
   Reason:
   - `src/libs/config.constant.ts` already derives theme options from `themes`.
   - Labels are auto-generated by `formatPresetThemeLabel`.

## Required updates

When adding a theme, update these tests:

- `src/utils/get-theme.spec.ts`
  Add assertions for the new theme palettes and label formatting when useful.

- `src/app.controller.spec.ts`
  Add an assertion that `/config` exposes the new theme option.

## Playground notes

The playground has two theme-related inputs:

- Theme metadata comes from `/config`.
- SVG previews are rendered locally through `@render-core`.

Because of that split, theme work can look stale in dev if only one process refreshed. The local preview path now prefers resolved `colors` from the active theme palette, which keeps the gallery and main preview aligned when themes are added or recolored.

If you change render logic itself, not just theme palettes, restart both dev servers:

- Nest server
- Vite playground server

## Validation checklist

Run these after adding or changing themes:

- `yarn test --runInBand -- app.controller.spec.ts utils/get-theme.spec.ts`
- `cd playground && yarn build`

Then manually verify in the playground:

- The new theme appears in the theme gallery.
- Dark and light mode both render correctly.
- The theme swatches match the actual SVG preview.
- The main preview no longer falls back to green.
